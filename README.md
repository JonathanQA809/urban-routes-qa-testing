# Urban Routes QA Testing Project

## Overview

This project focuses on manual testing of the Urban Routes web application. The goal was to identify bugs by executing predefined test cases and comparing expected results with actual behavior.

## Responsibilities

* Executed manual test cases based on predefined requirements
* Compared expected vs actual results to identify defects
* Reported bugs in Jira using structured bug reports
* Verified UI behavior and user flows

## Testing Types

* Functional Testing
* UI Testing
* Exploratory Testing

## Test Artifacts

* Test cases (pdf format)
* Bug reports (Markdown format)
* Screenshots of identified issues

## Tools Used

* Jira (bug tracking)
* Google Sheets (test cases)
* Chrome DevTools
* Browser: Microsoft Edge

## Key Outcome

* Identified multiple UI and functional defects
* Created clear, reproducible bug reports
* Strengthened understanding of manual QA processes

## How to Use This Repository

* View test cases in the `test-cases` folder
* Review bug reports in the `bug-reports` folder
* Check screenshots for visual evidence of bugs
